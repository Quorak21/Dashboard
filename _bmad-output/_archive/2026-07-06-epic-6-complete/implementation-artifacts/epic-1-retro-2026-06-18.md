## Challenges & Growth Areas

### 1. NPE Cluster in ConfigurableAssetService (Story 1.4 review)

**What happened:** Story 1.4 review flagged 5 null-guard issues in `ConfigurableAssetService` — null PriceQuote, null marketHours, null lastRefresh, null day in refreshHistory, null provider.

**Root cause:** Unit test fixtures were too "happy path". Null paths never exercised until review.

**Lesson:** Null-defensive testing must be proactive. Write at least one null/missing-field scenario per service test class.

---

### 2. `getData` serving stale `marketStatus` (Story 1.3 review)

**What happened:** `getData` was serving `marketStatus` frozen from cache instead of recalculating live via `MarketHoursGuard`. Patched in review.

**Root cause:** Spec said "`getData` never calls provider" but did not distinguish provider calls vs. derived state recalculation.

**Lesson:** For any cache-serving method, explicitly document which fields are served from cache vs. recalculated live.

---

### 3. Unbounded cache maps — silent memory risk

**What happened:** `ConfigurableAssetService` uses `ConcurrentHashMap<String, AtomicReference<AssetDto>>` with no eviction strategy. Deferred in Story 1.4 review.

**Why it matters for Epic 2:** Story 2.2 (`AssetSyncJob`) will call `syncPrice` on every registry asset every 15 minutes. Removed assets leave stale map entries permanently.

**Decision:** Fix before Story 2.2. Dokk identified this as the highest-priority debt item from Epic 1.

---

### 4. `IllegalArgumentException` too broad (Story 1.5 review)

**What happened:** Global handler caught all `IllegalArgumentException`. Resolved by creating dedicated `AssetNotFoundException`.

**Lesson:** Named exceptions from the start. No broad catches.

## Key Insights

1. Architecture discipline is the most valuable investment at this stage — full ADR compliance on Epic 1 means Epic 2 builds on solid ground.
2. The story spec format (scope IN/OUT tables, anti-pattern lists, file-level boundaries) significantly reduced ambiguity and scope drift.
3. Null-defensive testing is a habit, not a one-time action — must be enforced at story creation and review time.
4. Deferred debt must always be linked to a specific future story, not a vague "Epic futur".
5. The review-per-story process is working — real bugs caught, real decisions documented, real patterns surfaced.
